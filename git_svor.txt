
Git spurningar og svör

1. Hvað gera eftirfarandi Linux skipanir?
	
	Cd: breyta skrá 
	cd ~: Fer beint til baka (home directory) 
	ls: listi af upplysingum 
	pwd: Breytir um drif 
	mkdir: Býr til skrá 

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?

	git clone: klónar repository í nýtt directory
	git log: sýnir commit loggin
	git status: sýnir núverandi status
	git diff: sýnir muninn á milli skránna fyrir og eftir breytingar
	git checkout: Switch branches or restore working tree files
	
3. Hver er munurinn á eftirfarandi git skipunum: 
	a) git diff synir breytingar á skjali 
	b) git diff –staged Breytir bara skjali í „staged“ svæði 
	c) git diff commit1 commit2: munurinn á milli þeirra
	
4. Hvað er útgáfustýring (e.version control)?, 
Version control er ser utgafa af hjalpar tæki sem hjalpar forrtunar liði að halda utan um „source code“ yfir tima
Hverjir eru helstu kostir við að nota GIT? 
Það er að það er hægt að nota það þegar þu ert ekki nettengdur og það er fljott og auðvelt að vinna með það 

5. Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu? 
Margir commita eftir tima en mer finst það eiga vera gert þegar þu lagar eithvað staðsetninga villu eða þegar þu bætir við koða sem virkar 

6. Hvernig eru Working directory, Staging area og Repository frábrugðin í GIT, til hvers eru þau?
Working directory er mappan sem við erum að vinna í (í þessu tilfelli git_verkefni). Staging area er að gera skjalið tilbuið. Repository er geymslan.
